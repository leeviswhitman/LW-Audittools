# LW 审计工具箱

> 面向中国审计实务的 Excel 任务窗格插件，支持序时账处理、科目余额表分析、联查凭证、一键抽凭、勾稽校验与底稿输出。

## 功能概览

| 模块 | 功能说明 |
|------|----------|
| 📊 **项目概览** | 设置客户名称 / 审计期间 / 操作者；查看数据量统计、异常分录数、抽样数等 |
| 📥 **数据导入** | 自动扫描工作簿所有 Sheet，识别序时账、科目余额表、凭证明细等类型（高/中/低置信度）|
| 🔗 **字段映射** | 查看并调整自动识别的字段映射，确认后一键触发完整数据导入（清洗 → 解析 → 存储） |
| 🧹 **数据清洗** | 查看数据清洗日志：清洗规则汇总、错误明细、向下填充记录，支持全量展示 |
| 📋 **序时账分析** | 按科目发生额汇总（含月度趋势），一键展开各月明细；异常分录列表；借贷平衡验证 |
| ⚖️ **科目余额表** | 展示标准化余额表，含余额方向推断、期初期末数据、勾稽状态及异常标注 |
| 🔍 **联查凭证** | 按科目 / 凭证号 / 单据号穿透联查，展示关联分录、相似凭证、相邻日期冲销对 |
| 🎯 **一键抽凭** | 支持 6 种抽样方法：重大项目全选、Top N、随机、分层随机、固定间隔、风险导向抽样 |
| ✅ **勾稽校验** | 验证序时账借贷平衡与余额表的勾稽关系，输出差异明细 |
| 📄 **底稿输出** | 将审计分析结果一键写入新工作表，形成标准化审计底稿 |
| 📝 **审计日志** | 全程操作留痕，支持导出留档 |
| ⚙️ **系统设置** | 配置重要性水平、合计行关键词、风险摘要关键词、默认随机种子 |

## 快速安装

### 前提条件

- **Microsoft Excel 桌面版**（Windows 或 macOS），版本建议 Microsoft 365 / Excel 2019+
- 计算机需保持**联网状态**（插件前端托管于 GitHub Pages）

### 第一步：下载 manifest.xml

前往 [Releases 页面](https://github.com/leeviswhitman/LW-Audittools/releases/latest) 下载最新版的 **`manifest.xml`** 文件，保存到本地任意位置（如桌面）。

### 第二步：在 Excel 中旁加载插件

**Windows（Excel 桌面版）**

1. 打开 Excel，新建或打开任意工作簿
2. 点击顶部菜单 **"文件"** → **"选项"** → **"信任中心"** → **"信任中心设置"**
3. 在左侧选择 **"受信任的加载项目录"**，将以下路径加入受信任目录（或使用下面的上传方式）
4. 或者：点击顶部菜单 **"插入"** → **"加载项"** → **"我的加载项"** → **"管理我的加载项"** → **"上传我的加载项"**
5. 在弹出对话框中点击 **"浏览"**，选择下载的 `manifest.xml`，然后点击 **"上传"**

> 💡 如看不到"管理我的加载项"，请确认已启用"开发工具"选项卡：**文件 → 选项 → 自定义功能区 → 勾选"开发工具"**，然后通过 **"开发工具" → "Office 加载项" → "上传我的加载项"** 进行加载。

**macOS（Excel for Mac）**

1. 打开 Excel
2. 顶部菜单 **"插入"** → **"加载项"** → **"更多加载项"** → **"上传我的加载项"**
3. 选择下载的 `manifest.xml`，点击 **"上传"**

### 第三步：打开插件

加载成功后，Excel 的 **"开始"** 选项卡中会出现 **"LW 审计工具"** 按钮组，点击 **"打开审计工具箱"** 即可在右侧打开任务窗格。

## 使用指南

### 建议工作流程

```
① 设置项目信息（概览页）
② 导入数据（序时账 + 科目余额表）
③ 确认字段映射
④ 执行数据清洗
⑤ 序时账标准化分析
⑥ 余额表与序时账勾稽校验
⑦ 一键抽凭（选择抽样方法）
⑧ 联查凭证（追溯证据链）
⑨ 输出审计底稿
```

### 模块详细说明

#### 📊 项目概览

- 填写 **客户名称**、**审计期间**（如 `2023年度`）、**操作者姓名**
- 查看当前工作状态：已加载的分录数、科目数、凭证数、勾稽错误数、样本量

#### 📥 数据导入

1. 打开包含审计数据的 Excel 工作簿（可以是当前工作簿的其他 Sheet）
2. 点击 **"扫描工作簿"**，插件会自动识别每个 Sheet 的数据类型
3. 识别结果包含：类型（序时账 / 余额表 / 凭证明细等）及置信度（高 / 中 / 低）
4. 对于未识别的 Sheet，可在"字段映射"页面手动配置

> **序时账字段要求**：表头应包含日期、凭证号、摘要、科目代码/名称、借方金额、贷方金额等列，具体字段名称支持多种中文别名自动匹配。

#### 🎯 一键抽凭

支持以下 6 种抽样方法：

| 方法 | 适用场景 |
|------|----------|
| 重大项目全选 | 金额超过重要性水平的分录全部纳入 |
| Top N 金额抽样 | 取金额最大的 N 笔分录 |
| 随机抽样 | 从总体中均匀随机抽取 N 笔，支持固定随机种子保证可重现 |
| 分层随机抽样 | 按金额区间分层后分别随机抽取 |
| 固定间隔抽样 | 每隔 N 条抽取 1 条（系统抽样）|
| 风险导向抽样 | 优先抽取：期末分录、敏感摘要（返点/折扣等）、非工作日、手工凭证、冲销凭证 |

操作步骤：
1. 先完成数据导入
2. 选择抽样方法并配置参数
3. 点击 **"执行抽样"**
4. 点击 **"导出《抽凭样本清单》到新 Sheet"** 输出结果

#### ✅ 勾稽校验

- 验证序时账各科目发生额加减期初余额后，是否与科目余额表期末余额一致
- 输出差异汇总：错误数量、差异金额、错误明细列表

#### 📄 底稿输出

- 将抽凭清单、勾稽结果等写入当前工作簿的新 Sheet
- 新 Sheet 命名格式：`抽凭样本清单_YYYYMMDD`

#### 📝 审计日志

- 自动记录所有数据导入、抽样、校验、导出操作
- 支持导出日志到新 Sheet，便于留档

## 本地开发

### 环境要求

- Node.js 20+
- npm 9+

### 启动开发服务器

```bash
npm install
npm run dev
```

开发服务器默认启动在 `https://localhost:3000`（HTTPS）。

> ⚠️ Office Add-in 要求 HTTPS。首次运行时需要在 Excel 中信任本地自签名证书，或配置 `vite.config.ts` 使用自定义证书。

### 在 Excel 中加载开发版本

使用根目录的 `manifest.xml`（指向 `https://localhost:3000`），按[快速安装](#快速安装)的步骤旁加载即可。

### 构建生产版本

```bash
npm run build
```

构建产物输出到 `dist/` 目录。如需部署到 GitHub Pages：

```bash
VITE_BASE_URL=/LW-Audittools/ npm run build
```

### 运行测试

```bash
npm run test
```

### 发布新版本

推送以 `v` 开头的 tag（如 `v1.0.1`）即可自动触发 GitHub Actions：

```bash
git tag v1.0.1
git push origin v1.0.1
```

工作流会自动：
1. 构建项目并部署到 GitHub Pages
2. 创建 GitHub Release 并附上 `manifest.xml`

## 注意事项

1. **数据安全**：本插件完全在客户端（浏览器）运行，不会将任何数据上传至外部服务器。
2. **审计责任**：所有自动分析结果（抽样、勾稽、异常识别等）仅为辅助工具，需由注册会计师进行独立的职业判断与复核，不能替代审计程序。
3. **浏览器兼容性**：基于 Office.js ExcelApi 1.9+，支持 Microsoft 365 及 Excel 2019+。
4. **联网要求**：使用正式版（GitHub Pages 托管）时需保持网络连接，以加载插件前端页面及 Office.js 库。
